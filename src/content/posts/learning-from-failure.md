每个人都犯过错。分享几个我犯过的。

## 1. 盲目追新技术

大二的时候，一个简单的课程项目非要用 K8s 部署。结果花了两周配置环境，代码写了半天。最后交作业时因为 K8s 出了问题，差点没过。

教训：选择最简单的能解决问题的方案。一个 jar 包 + 一台服务器能把事做了，就别上 K8s。

## 2. 依赖注入上瘾

学 Spring 之后，什么都要用依赖注入。一个不到 500 行的 CLI 工具，写了 3 个接口、5 个实现类。

教训：如果 new 一下就能解决的事，就别搞 DI。不是所有代码都需要「企业级」。

## 3. 注释写了等于没写

```java
// 获取用户
public User getUser(Long id) { ... }
```

这样的注释毫无价值。

教训：好代码不需要注释来解释「做什么」。注释应该解释「为什么」——为什么用这个算法？为什么不考虑缓存？为什么用 30 秒的超时？

## 4. 过早学太多

有一段时间既想学 Rust 又想学 K8s 还想学机器学习。同时开三个方向，最后哪个都没学透。

教训：一段时间专注一件事。深度的价值远大于广度。

犯错不可怕。可怕的是一遍遍犯同样的错。
