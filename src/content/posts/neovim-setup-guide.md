用了一年 Neovim，分享一下配置心得。

## 为什么选 Neovim

- 启动快（50ms vs Code 的 5s）
- 内存占用低（200MB vs Code 的 1GB+）
- 纯键盘操作，不需要鼠标
- 配置文件就是代码，可以版本控制
- 在远程服务器上也能用

## 核心插件

我用的插件不多，但都很关键：

- Lazy.nvim：插件管理器（替代 Packer）
- nvim-lspconfig：LSP 配置（代码补全、跳转定义、错误提示）
- nvim-cmp：自动补全引擎
- Telescope：模糊查找（文件、符号、Git）
- neo-tree：文件浏览器
- gitsigns：Git 状态显示
- tokyonight：配色主题

## 配置思路

不要把别人的配置直接搬过来。最好的方式是：
1. 从最少的插件开始
2. 遇到痛点再找解决方案
3. 理解每个插件的配置
4. 定期清理不用的插件

## VS Code 用户要转吗？

- 如果你经常在远程服务器上写代码 → 强烈推荐
- 如果你主要写 Java/C# → 算了（IDE 更好）
- 如果你写 Python/Go/Rust/JS → 值得尝试
- 如果你追求「手不离键盘」的体验 → 必须试

Neovim 的学习曲线确实陡，但一旦适应了，就回不去了。
