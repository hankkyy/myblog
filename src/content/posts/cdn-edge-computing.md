边缘计算（Edge Computing）是 CDN 的进化版——不仅缓存静态文件，还能在离用户最近的节点上运行代码。

## 为什么需要边缘计算

传统架构：用户 → CDN（静态） → 源站（动态处理）

边缘计算：用户 → 边缘节点（动态 + 静态都在这里处理）

一个用户在新加坡，源站在美国。传统模式下动态请求要跨半个地球（200ms+）。边缘计算模式下直接在最近节点处理（10ms）。

## 适用场景

- A/B 测试（在边缘根据 cookie 返回不同版本）
- 图片优化（实时缩放、转格式）
- API 网关（认证、限流、路由在边缘完成）
- 地理位置相关的逻辑
- 简单的服务端渲染

## 不适合的场景

- 需要数据库的操作（延迟和不一致问题）
- 计算密集型任务（边缘节点的 CPU 有限）
- 有状态的逻辑（边缘节点之间不共享状态）

## 主流平台

- Cloudflare Workers：全球最大边缘网络，支持 JS/TS/WASM
- Vercel Edge Functions：和 Next.js 深度集成
- Deno Deploy：基于 Deno 的边缘平台
- AWS Lambda@Edge：跟 CloudFront 绑定

边缘计算是一个重要的趋势——不是替代传统的后端，而是让某些关键路径变得更快。
