开源项目从 0 到 1000 Star 的真实复盘

维护开源项目一年后，分享一些没人告诉你的真相。

前 100 个 Star 最难

第一个月，每天刷新 GitHub 几十次。Issue 比 Star 多。

破局点：在 Reddit / V2EX / Twitter 发了三篇长文介绍项目。带来前 200 个 Star。

README 比代码重要

用户第一眼看到的是 README：

  • 3 秒决定是否继续看
  • 要有截图/GIF
  • 5 行内说清楚这个项目做什么
  • 快速开始 < 3 个命令

Issue 是免费的产品经理

早期用户提的 Issue 价值连城。每一条都是真实需求。

学到了： - 24 小时内回复每个 Issue - 不要怕说「不」 - good first issue 标签吸引贡献者

中文社区 vs 英文社区

英文社区反馈更友好，但中文社区传播更快。

建议：README 中英双语，Issue 用英文。

真实数据

第 1 周:    3 Star
第 1 月:   50 Star
第 3 月:  200 Star（V2EX 热帖）
第 6 月:  500 Star（GitHub Trending）
第 12 月: 1200 Star

开源不是写代码，是做产品。
